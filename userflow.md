# Customer Registry - User Flow

## Introduction

The Customer Registry application provides a simple and organized workflow for managing customer information and handling customer complaints. The system involves three main users:

- Customer
- Admin
- Agent

Each user performs specific actions to ensure smooth communication and efficient complaint resolution.

# Step 1: User Registration

The customer starts by creating a new account.

### Process

- Open the Customer Registry application.
- Click **Register**.
- Enter personal information such as:
  - Full Name
  - Email Address
  - Mobile Number
  - Password
- Submit the registration form.
- The system validates the entered details.
- If successful, the account is created.
- The customer receives a confirmation message.

---

# Step 2: User Login

After registration, the customer logs into the application.

### Process

- Open the application.
- Enter registered email and password.
- Click **Login**.
- The system verifies the credentials.
- If valid, the customer is redirected to the dashboard.

---

# Step 3: Customer Dashboard

The dashboard provides quick access to all customer services.

Customers can:

- View profile
- Update profile information
- Raise complaints
- View complaint history
- Check complaint status
- Provide feedback
- Logout

---

# Step 4: Raise a Complaint

The customer reports an issue through the application.

### Process

- Select **Raise Complaint**.
- Enter complaint details.
- Attach supporting documents if required.
- Submit the complaint.
- The system generates a unique Complaint ID.
- A confirmation message is displayed.

---

# Step 5: Admin Review

The administrator manages all incoming complaints.

### Responsibilities

- View newly submitted complaints.
- Verify complaint details.
- Categorize the complaint.
- Assign the complaint to an available support agent.
- Notify the assigned agent.

---

# Step 6: Agent Resolution

The assigned support agent handles the complaint.

### Process

- View assigned complaint.
- Contact the customer if additional information is required.
- Investigate the issue.
- Update complaint progress.
- Resolve the complaint.
- Mark complaint as **Resolved**.

---

# Step 7: Customer Feedback

After resolution, the customer receives a notification.

The customer can:

- Review the solution.
- Rate the support service.
- Submit feedback.
- Complete a satisfaction survey if available.

The system stores the feedback and updates the complaint status to **Closed**.
